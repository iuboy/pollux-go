# gosec 配置说明

本文件说明根目录 `.gosec.json` 中每条排除规则的排除理由。gosec 的 JSON 配置不支持注释,
因此排除理由记录在此文档中。

## 配置总览

```json
{
  "nosec": { "enabled": true },
  "exclude": ["G402", "G304", "G115", "G401", "G405", "G501", "G502", "G505"]
}
```

### `nosec.enabled: true`

允许源码中使用 `//nolint:gosec` / `#nosec` 标注对特定误报进行定向豁免。本项目仅在以下
场景使用:上游库已知误报(如 `sm2/key_exchange.go` 中保留上游拼写错误的标注)、以及已通过
密码学审计确认安全的代码路径。每个 `#nosec` 标注都附有注释说明豁免理由。

## 排除的规则

> **原则**:密码学库需要操作原始密钥字节、文件输入、弱遗留算法以实现互通性,gosec 针对普通
> 应用代码的通用规则在此场景下产生大量误报。每类风险通过**输入校验、测试向量、审计**而非
> 规则告警来控制。详见 `docs/plan/AUDIT_REPORT.md`。

| 规则 | 含义 | 排除理由 |
|------|------|----------|
| **G402** | TLS `InsecureSkipVerify` / 弱 TLS 配置 | 本项目实现 TLCP/TLS 协议栈本身,需在受控测试中显式设置 `InsecureSkipVerify`、自定义 cipher suite、最小版本等。这些是协议测试的合法需求,真实客户端的证书校验由调用方负责。 |
| **G304** | 文件路径潜在路径遍历(`os.ReadFile(tainted)`) | 密钥/证书加载函数(`LoadX509KeyPair`、`os.ReadFile` 等)的路径来自程序配置或可信输入,非外部用户可控。文件读取由 `os.Open` 的权限语义保护。 |
| **G115** | 整数类型转换可能溢出 | 密码学常量、长度字段、序列号等处存在大量 `int`↔`uint`↔`uint16` 转换,值域经协议规范约束且已校验。 |
| **G401** | 弱哈希 MD5 | TLCP/证书互通层需兼容部分遗留格式解析路径;核心摘要一律使用 SM3。 |
| **G405** | 弱哈希 SHA1 | 同 G401,遗留证书/互通性解析需要。 |
| **G501** | 弱加密导入 DES | 互通性与遗留 PEM/密钥格式解析需要。 |
| **G502** | 弱加密导入 RC4 / ChaCha 黑名单误报 | 依赖链中的密码学包导入触发;核心对称加密一律使用 SM4-GCM/CCM。 |
| **G505** | 弱加密导入 SHA1/DES 区块 | 同 G401/G501,遗留互通性需要。 |

## 审计替代

排除上述通用规则**不等于**放弃安全管控。本项目的密码学安全性由以下机制保证:

1. **专项密码学审计**:`docs/plan/AUDIT_REPORT.md` 记录 15 个 CRITICAL + 32 个 HIGH 问题,
   全部修复(见 `docs/plan/audit-status-matrix.md`)。
2. **回归测试矩阵**:每个审计修复项映射到具体的回归测试(见审计状态矩阵"回归测试名映射")。
3. **Fuzz 测试**:`tlcp/fuzz_test.go`、`smx509/fuzz_test.go` 覆盖握手消息、记录层、证书解析、
   解密、padding 校验等解析路径。
4. **内存安全**:`internal/memsecure` 提供密钥材料清零机制,防止编译器优化。

## 运行 gosec

```bash
gosec -conf .gosec.json ./...
```
