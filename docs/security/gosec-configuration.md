# gosec 配置说明

说明根目录 `.gosec.json` 每条排除规则的排除理由。gosec 的 JSON 配置不支持注释，因此记录在此。

## 配置

```json
{
  "nosec": { "enabled": true },
  "exclude": ["G402", "G304", "G115", "G401", "G405", "G501", "G502", "G505"]
}
```

`nosec.enabled: true` 允许源码用 `//nolint:gosec` / `#nosec` 对特定误报定向豁免，每个标注都附注释说明理由。

## 排除的规则

密码学库需要操作原始密钥字节、文件输入、弱遗留算法以实现互通性，gosec 针对普通应用代码的通用规则在此场景下误报较多。每类风险通过输入校验、测试向量、回归测试控制，而非依赖规则告警。

| 规则 | 含义 | 排除理由 |
|------|------|----------|
| **G402** | TLS `InsecureSkipVerify` / 弱 TLS 配置 | 本项目实现 TLCP/TLS 协议栈本身，受控测试中需显式设置这些项；真实客户端的证书校验由调用方负责。 |
| **G304** | 文件路径遍历（`os.ReadFile(tainted)`） | 密钥/证书加载路径来自程序配置或可信输入，非外部用户可控。 |
| **G115** | 整数类型转换溢出 | 密码学常量、长度字段、序列号等转换值域经协议规范约束且已校验。 |
| **G401** | 弱哈希 MD5 | TLCP/证书互通层需兼容遗留格式解析；核心摘要一律用 SM3。 |
| **G405** | 弱哈希 SHA1 | 同 G401，遗留证书/互通性解析需要。 |
| **G501** | 弱加密导入 DES | 互通性与遗留 PEM/密钥格式解析需要。 |
| **G502** | RC4 / ChaCha 黑名单误报 | 依赖链密码学包导入触发；核心对称加密一律用 SM4-GCM/CCM。 |
| **G505** | SHA1/DES 区块 | 同 G401/G501，遗留互通性需要。 |

## 安全保障

排除上述通用规则不等于放弃安全管控。密码学安全性由以下机制保证：

1. **回归测试**：关键路径映射到具体的回归测试。
2. **Fuzz 测试**：`tlcp/fuzz_test.go`、`smx509/fuzz_test.go` 覆盖握手消息、记录层、证书解析、解密、padding 校验等路径。
3. **内存安全**：`internal/memsecure` 提供密钥材料清零机制。

## 运行

```bash
gosec -conf .gosec.json ./...
```
